3.3 Работа с CSV файлами

 
Defining a CSV connection and dataset
 
Для работы с CSV-файлами необходимо создать соединение и набор данных:
 
def csvfile = csv {
  // Create and set connection for work
  useConnection csvConnection {
    // File storage path
    path = '/tmp'
    // Create path if missing
    createPath = true
  }
 
  // File name
  fileName = 'file1'
 
  // Add the extension "csv" to the file name
  extension = 'csv'
 
  // Separator of field values in a record
  fieldDelimiter = ','
 
  // Row record separator
  rowDelimiter = '\r\n'
 
  // Sets the quote character within which to enclose data
  quoteStr = '"'
 
  // Quote text fields that have quotation marks or line feeds
  quoteMode = quoteNormal
 
  // Character set (use standard Java charset names)
  encodePage = 'utf-8'
 
  // Compressing the file to GZ is not required
  isGzFile = false
 
  // The first line of the file contains the header of the field list.
  header = true
 
  // Adding data to the end of an existing file is not required
  append = false
 
  // Delete a file if it appears without rows after writing data to it
  deleteOnEmpty = true
 
  // Check constraints (PK/NOT NULL) during reading and writing file
  constraintsCheck = true
 
  // Write a file in RFC 4180 format (otherwise the file will be written in "escaping" format)
  escaped = false
 
  // Interpret the specified value as NULL
  nullAsValue = '!NULL!'
 
  // Use the specified locale to work with date and number fields
  locale = 'en-EN'
 
  // Use a dot delimiter to read and write numeric fields
  decimalSeparator = '.'
 
  // Use the specified mask for reading and writing date fields
  formatDate = 'yyyy-MM-dd'
 
  // Use the specified mask for reading and writing timestamp fields
  formatDateTime = 'yyyy-MM-dd HH:mm:ss.SSS'
 
  // Use the specified mask for reading and writing time fields
  formatTime = 'HH:mm:ss.SSS'
 
  // File fields
  field('id') { type = integerFieldType }
  field('name')
  field('dt') { type = dateFieldType }
}
 
Если вы хотите работать с большим количеством файлов с одинаковой конфигурацией, их свойства можно установить в свойствах соединения, а не в свойствах файла.
 
 
Использование временных файлов CSV
 
Временные файлы позволяют работать с промежуточными результатами и будут автоматически удалены в конце работы приложения. Таким файлам не нужно устанавливать соединение и имя файла, они будут располагаться по пути хранения временных файлов в ОС:
 
def tempfile = csvTemp {
  fileName = 'file1'
 
  field('id') { type = integerFieldType }
  field('name')
  field('dt') { type = dateFieldType }
}
 
Создание файлов со структурой аналагичной другому источнику
 
Вы можете просто скопировать список полей между различными наборами данных:
 
def mstable = mssql {
  tableName = 'table1'
  field('id') { type = bigintFieldType; isKey = true }
}
 
def csvfile = csv {
  fileName = 'file1.csv'
  field = mstable.field
}
 
 
Вы можете создать описание нового файла CSV на основе источника:
 
def mstable = mssql {
  tableName = 'table1'
  field('id') { type = bigintFieldType; isKey = true }
}
csvfile = csvWithDataset(mstable) {
  fileName = 'file1.csv'
}
tempfile = csvTempWithDataset(mstable) {
  fileName = 'file1.csv'
}
 
 
Параметры файла CSV
 
Для файлов вы можете установить параметры чтения и записи:
 
csv {
  fileName = 'file1'
  extension = 'csv'
 
  readOpts {
    // Read all files by given mask as "file1.{num}.csv"
    isSplit = true
  }
  writeOpts {
    // When recording, split into separate files of 1 mb in size, assigning names to files by mask "file1.{num}.csv"
    splitSize = 1000000
  }
}
 
 
При записи большого объема данных в файлы можно указать разделение при записи файлов на порции необходимого размера.
 
При записи порций к имени файла добавляется 4-значный номер порции. Вы также можете сами управлять порциями при записи, указав код для «splitFile»:
 
 
csv {
  fileName = 'file1'
  extension = 'csv'
 
  field('id') { type = integerFieldType }
  field('name')
  field('dt') { type = dateFieldType }
 
  writeOpts {
    // Write a file in portions for each thousand id field values
    splitFile { row ->  (row.id as Integer).mod(1000) == 0 }
 
    // A portion of the file is immediately readable after the start of writing another portion
    avaibleAfterWrite = true
  }
}
