3.2.2 Работа с RDBMS таблицами

Описание Таблицы
 
Для работы с таблицей бд необходимо указать соединение с СУБД и задать параметры:
 
def oracon = oracleConnection {
  connectURL = 'jdbc:oracle:thin:@localhost:1521:demo'
  login = 'user'
  password = 'password'
}
 
oracleTable {
  useConnection oracon // set connection to "oracon"
  schemaName = 'demo'
  tableName = 'table1'
}
 
 
Получение списка полей
 
Если таблица существует в базе данных, то при работе с ней будет автоматически получен список ее полей. Если нужно явно получить список полей, используйте метод «retrieveFields»:
 
def oratab = oracleTable {
  useConnection oracon
  schemaName = 'demo'
  tableName = 'table1'
  assert field.size() == 0
  retrieveFields()
  assert field.size() > 0
}
 
Проверка наличия таблицы в базе данных
 
Метод "exists" позволяет проверить, существует ли указанная таблица в базе данных
 
oratab.with {
  if (!exists) logWarning("Table ${fullNameDataset()} not define in database!")
}
 
 
Создание таблицы в базе данных
 
Если таблицы нет в базе данных и вам нужно ее создать, можно явно описать поля и вызвать создание таблицы:
 
oratab.with {
  field('id') { type = integerFieldType; isKey = true }
  field('name') { type = stringFieldType; length = 50; isNull = false }
  assert !exists
  create()
  assert exists
}
 
 
Удаление таблицы из базы данных
 
Метод "drop" позволяет удалить таблицу в базе данных:
 
oratab.with {
  assert exists
  drop()
  assert !exists
}
 
Полная очистка таблицы
 
Метод "truncate" удаляет все записи в таблице. По умолчанию вызывается оператор "DELETE FROM table", может быть передан параметр "truncate: true" тогда очистка будет произведена с помощью оператора "TRUNCATE TABLE table":
 
oratab.with {
  truncate(truncate: true)
  assert countRow() == 0
}
Table options
When describing table parameters, you can specify options for creating, reading, writing, and bulk loading:
 
def vertable = verticaTable {
  useConnection verticaConnection {
    connectHost = 'localhost'
    connectDatabase = 'demo'
    login = 'user'
    password = 'password'
  }
 
  schemaName = 'demo'
  tableName = 'table1'
 
  field('id') { type = integerFieldType; isKey = true }
  field('name') { type = stringFieldType; lenth = 100; isNull = false }
  field('dt') { type = dateFieldType; isNull = false }
 
  createOpts {
    orderBy = ['dt', 'id']
    segmentedBy = 'hash(id) all nodes'
    partitionBy = 'dt'
    ifNotExists = true
  }
 
  dropOpts {
    ifExists = true
  }
 
  readOpts {
    label = 'my query hint'
  }
 
  writeOpts {
    direct = 'AUTO'
  }
 
  bulkLoadOpts {
    compressed = 'GZIP'
    loadMethod = 'DIRECT'
    enforceLenth = true
  }
}
 
 
Массовая загрузка файлов в таблицу (Bulk load)
 
Для DBMS H2 базы данных, Hadoop Hive и Microfocus Vertica, поддерживается batch загрузка из файлов CSV в таблицу.
Для этого нужно вызвать команду «bulkLoadCsv», передав ей параметр «csv» объекта, который задает имя файла, его параметры парсинга и указывает путь к хранилищу файлов в соединении. Если вместо одного файла нужно загрузить несколько по маске или списку файлов, следует использовать опцию "files" option:
 
def csv = csv {
  useConnection csvConnection { path = '/tmp/csv' }
  fileName = 'data1.csv'
}
 
vertable.with {
  // Load single file
  bulkLoadCsv(csv)
 
  // Load list of files
  bulkLoadCsv(csv) {
    files = ['data1.csv', 'data2.csv']
  }
 
  // Load files by simple mask where all the files are loaded in one command and not one file (only for Hive and Vertica tables)
  bulkLoadCsv(csv) {
    files = 'data*.csv'
    loadAsPackage = true
  }
 
  // Load files by advanced mask where the loading order is determined by sorting by the macro field "num"
  bulkLoadCsv(csv) {
    files = main.filePath {
      mask = 'data{num}.csv'
      variable('num') { type = integerFieldType }
    }
    orderProcess = ['num']
 
    // Custom code before file load
    beforeBulkLoadFile { println 'before: ' + it }
 
    // Custom code after file load
    afterBulkLoadFile { println 'after: ' + it }
  }
}
