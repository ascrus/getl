3.2.3 Выполнение скриптов SQL для источников JDBC

 
Объект "query"
 
Для работы с данными запроса, полученными с использованием запроса SQL, используется объект "query":
 
def qryFact = query {
  // Use Netezza connection
  useConnection netezzaConnection { useConfig 'netezza' }
 
  // SQL query
  query = '''
SELECT fact.*, dim.name AS dim_name
FROM fact
  INNER JOIN dim ON fact.dim_id = dim.id
WHERE
  fact.dt >= CAST('{fact_date}' AS date) AND
  dim.price BETWEEN {min_value} AND {max_value}
'''
 
  // Setting value to parameters
  queryParams.min_value = 100
  queryParams.max_value = 200
  queryParams.fact_date = '2019-12-31'
}
 
// Create csv temporary file
def csvFact = csvTemp()
 
// Copy from a query to a csv temporary file with an identical structure
copyRows(qryFact, csvFact) { inheritFields = true }
 
 
 
В скриптах запросов вы можете использовать переменные в фигурных кавычках, значения которых указаны в "queryParams".
 
Вместо описания текста запросов в коде они могут быть получены в виде файлов или ресурсных файлов, вызывая загрузку их содержимого:
 
query {
  // Use Netezza connection
  useConnection netezzaConnection { useConfig 'netezza' }
 
  // Load from resource file
  loadFile 'resource:/sql/fact_query.sql'
 
  // Setting value to parameters
  queryParams.min_value = 100
  queryParams.max_value = 200
  queryParams.fact_date = '2019-12-31'
 
  // Printing query rows
  eachRow { println it }
}
 
 
Оператор "sqlQuery"
 
По аналогии с использованием "query", можно работать со строками запроса, используя оператор "sqlQuery", для которого указывается соединение, текст запроса и его параметры:
 
// Setting default Netezza connection
useNetezzaConnection netezzaConnection { useConfig 'netezza' }
 
// Create csv temporary file
def csvFact = csvTemp()
 
copyRows(
  sqlQuery(textFromFile('resource:/sql/fact_query.sql'),
           [min_value: 100, max_value: 200, fact_date: '2019-12-31']), csvFact) {
    inheritFields = true
}
 
Функция "textFromFile" возвращает текст указанного файла или ресурсного файла.
 
 
 
 
Оператор "sqlQueryRow"
 
Чтобы быстро получить значения первой записи запроса, используйте оператор "sqlQueryRow":
 
println "first row: " +
  sqlQueryRow(netezzaConnection { useConfig 'netezza' },
    textFromFile('resource:/sql/fact_query.sql'),
    [min_value: 100, max_value: 200, fact_date: '2019-12-31']
  )
 
 
Оператор "sql"
 
Для выполнения пакета команд SQL используется оператор «sql»:
 
sql {
  // Use Netezza connection
  useConnection netezzaConnection { useConfig 'netezza' }
 
  // Run the script by transferring it completely to the server
  exec false, '''
CREATE TABLE test (
  id int NOT NULL,
  name varchar(50) NOT NULL
);
 
INSERT INTO test (id, name) VALUES (1, 'test 1');
COMMIT;
'''
 
  // Setting script variables
  vars.id = 1
 
  // Run the script using the Getl stored procedure language parser
  exec '''
-- Check condition
IF (EXISTS(SELECT * FROM test WHERE id = {id}));
  -- Get the value of row fields in variables
  SET SELECT name FROM test WHERE id = {id};
  ECHO The name field contains an "{name}" for identifier {id}.
END IF;
'''
 
  // Run a script from the specified file or resource file
  runFile 'resource:/sql/script.sql'
}
 
 
 
Работа с переменными в процедурном языке Getl
 
Работа в скрипте с переменными:
 
SELECT * FROM table WHERE field1 = {var1} AND field2 = '{var2}';
SELECT * FROM {var_table};
SELECT {var_field1}, {var_field2} FROM table;
 
Сохранить в переменной количество обработанных записей в скрипте:
 
/*:var_count*/
INSERT INTO table1 AS SELECT * FROM table2;
ECHO {var_count} rows inserted
IF ({var_count} = 0);
  EXIT;
END IF;
 
 
Команды sql процедурного языка Getl
 
Отправить сообщение в лог:
 
ECHO <message>
 
 
Выполните код, если условие истинно:
 
IF (condition);
  <sql code>
END IF;
 
 
Получить значения полей первой записи в переменные:
 
SET
  SELECT field AS var_name, ... FROM ...;
 
 
Выполните код для каждой строки заданного SQL-запроса:
 
FOR
  SELECT field AS var_name, ... FROM ...;
  <sql code>
END FOR;
 
 
Выполнить блок кода без предварительного парсинга:
 
BEGIN BLOCK;
  <sql code>
END BLOCK;
 
 
 
Сохранить значение чисел или дат в инкрементальной таблице истории:
 
SAVE_POINT [<db name>.][<schema name>.]<table name> FROM <variable name> WITH <MERGE|INSERT>;
 
 
 
Получить последнее сохраненное инкрементальное значение числа или даты из таблицы истории в переменную:
 
LOAD_POINT [<db name>.][<schema name>.]<table name> TO <variable name> WITH <MERGE|INSERT>;
 
 
 
окончание скрипта:
 
EXIT;
 
 
Завершить скрипт с ошибкой, написав сообщение в лог:
 
EXIT <Error message>;
