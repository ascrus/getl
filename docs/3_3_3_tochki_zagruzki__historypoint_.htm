3.3.3 Точки загрузки (Historypoint)

Точки загрузки позволяют организовать инкрементальные выгрузки данных. Для этого они могут фиксировать последние значения выгружаемых данных с источников и использовать эти значения в фильтрах при последующих выгрузках новых порций данных. Разрешается использовать точки загрузки для контроля числовых полей и полей даты-времени. Значения точек загрузки хранятся в таблице РСУБД, где для точки загрузки можно указать соединение к БД и имя таблицы для хранения информации.
 
Доступ к объектам набора данных происходит по их имени в репозитории. Без указания имени создаются локальные объекты.
 
Для описания точки загрузки используют оператор регистрации в репозитории. Пример регистрации точки загрузки в СУБД Vertica с хранением данных в таблице public.history_points.
historypoint('dwh:history', true) {
    useConnection verticaConnection('dwh:con')
    schemaName = 'public'
    tableName = 'history_points'
    saveMethod = insertSave
    create true
}
 
В параметре "saveMethod" указывается способ обновления новой информации по таблице. Для OLTP СУБД наиболее эффективным является метод "mergeSave", который вызыввает UPDATE при обновлении данных точки загрузки для выгружаемой таблицы. Для Vertica более эффективным является метод "insertSave", который ведет для точки загрузки историю всех изменений, добавляя новые записи и выбирая при выборе последней точки загрузки максимальное значений из записей.
 
Если при регистрации точки загрузки указывается create, то в БД автоматически создается таблица с указанным именем, где параметр true указывает, что создавать таблицу нужно только, если она отсутствует. Рекомендуется вынести создание таблицы точки загрузки в отдельный инициализационный скрипт, убрав с скрипта регистрации точки загрузки в репозитории. Это позволит избежать лишних DDL запросов к БД и позволит более точно контролировать развертывание решения на новом стенде.
 
Для сохранения точки загрузки используется метод "saveValue", в котором указывается имя таблицы или обьекта, для которой сохраняется значение точки загрузки:
historypoint('dwh:history') {
    saveValue 'table1', DateUtils.now
}
 
Для чтения точки загрузки используется метод "lastValue", в котором указывается имя таблицы или объекта, для которого считывается значение последней точки загрузки:
historypoint('dwh:history') {
    println lastValue('table1')
}
 
Для очистки точки загрузки по конкретной таблице/объекту используйте метод "clearValue":
historypoint('dwh:history') {
    clearValue 'table1'
}
 
Для очистки точек загрузок всех таблиц используйте метод "truncate":
historypoint('dwh:history') {
    truncate()
}
 
Удаление таблицы, в которой хранятся точки загрузки возможно сделать с помощью метода "drop", параметр которого позволяет удалить только существующую таблицу.
