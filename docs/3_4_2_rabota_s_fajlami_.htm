3.4.2 Работа с файлами

fileCopier
 
"fileCopier" это процесс, который позволяет копировать файлы по заданной маске из одной файловой системы в другую.
 
Чтобы определить список файлов для копирования, вам необходимо указать маску в свойстве «sourcePath».
 
Чтобы задать фильтр, ограничивающий какие каталоги проверять на предмет копирования, укажите свой код в «filterDirs».
Чтобы задать фильтр, определяющий какие файлы следует копировать, укажите свой код в «filterFiles».
 
Если во время копирования требуется изменить путь к месту назначения, его маску можно установить в свойстве "destinationPath".
 
Если вы хотите изменить имя файла при копировании, вы можете установить маску переименования в свойстве "renamePath".
 
Чтобы удалить файлы и пустые каталоги в источнике после копирования, используйте свойства «removeFiles» и «removeEmptyDirs».
 
Если при описании источника указана таблица истории в свойстве «story», то при определении списка файлов для копирования, хранящиеся в таблице файлы исключаются из списка.
При копировании файлов их имена будут добавлены в эту таблицу.
 
Копирование будет происходить в соответствии с сортировкой, указанной в "order" (по умолчанию будут взяты путь и имена файлов).
 
Если "destination" определено как список файловых менеджеров, а имена полей определены в списке "segmented", тогда при копировании файл будет распределен между указанными получателями с группировкой по указанным полям распределения.
 
def t = h2table { tableName = 'story_copy' }
def s = files { rootPath = '/data'; useStory t }
def d = sftp { rootPath = '/tmp/data'; useConfig 'test' }
fileCopier(s, d) {
  useSourcePath {
    mask = '{dir}/{subdir}/file_{date}.{num}'
    variable('date') { type = datetimeFieldType }
    variable('num') { type = integerFieldType }
  }
  useDestinationPath { mask = '{dir}' }
  useRenamePath { mask = '{subdir}_file_{date}.{num}' }
  order = ['date', 'num']
  removeFiles = false
  removeEmptyDirs = false
}
 
 
fileCleaner
 
Процесс «fileCleaner» удаляет все файлы из источника в соответствии с указанной маской.
 
Чтобы задать список файлов, которые нужно удалить, нужно указать маску в свойстве «sourcePath».
 
Чтобы задать фильтр, определяющий какие каталоги проверять на удаление, следует указать свой код в «filterDirs». Чтобы определить фильтр, какие файлы удалять, указать своё условие в «filterFiles».
 
def s = sftp { rootPath = '/tmp/data'; useConfig 'test' }
fileCleaner(s) {
  useSourcePath {
    mask = '{dir}/{subdir}_file_{date}.{num}'
    variable('date') { type = datetimeFieldType }
    variable('num') { type = integerFieldType }
  }
  filterDirs { Map dir -> dir.filename != 'test dir' }
  filterFiles { Map file -> file.num == 1 && file.date = DateUtils.CurrentDate() }
  removeFiles = true
  removeEmptyDirs = true
}
 
 
fileProcessing
 
Процесс «fileProcessing» позволяет организовать многопоточную обработку исходных файлов по заданной маске.
 
Чтобы задать список файлов для обработки, нужно указать маску в свойстве «sourcePath».
 
Чтобы задать фильтр, определяющий какие каталоги следует проверять, нужно указать свой код в «filterDirs». Чтобы задать фильтр, файлов для обработки, следует указать свой код в «filterFiles».
 
If the source indicates the table in "story" property, then when defining the processing list, files that are stored in the history table are excluded from it. When processing files, their names will be added to this table.
 
Если  в описании источника указана таблица истории в свойстве «story», то при определении списка обработки хранящиеся в таблице историифайлы исключаются из списка. При обработке файлов их имена будут добавлены в таблицу истории.
 
Обработка будет происходить в соответствии с сортировкой, указанной в списке "order"  (по умолчанию будут взяты путь и имена файлов).
 
If uniform processing of files by specified groups is required, specify the grouping fields in property "threadGroupColumns" (by default, "filepath" is used to group processing). In parameter "countOfThreadProcessing" you can set the number of simultaneous file processing threads.
 
Если требуется равномерная обработка файлов по заданным группам, укажите поля группировки в свойстве «threadGroupColumns» (по умолчанию для групповой обработки используется «filepath»). В параметре «countOfThreadProcessing» вы можете установить количество потоков одновременной обработки файлов.
 
Чтобы удалить файлы и пустые каталоги в источнике после обработки, используйте свойства «removeFiles» и «removeEmptyDirs».
 
Вы можете сохранить обработанные файлы в указанном источнике, указанном в свойстве «storageProcessedFiles». Также можно сохранять файлы, при обработке которых произошла ошибка в источнике, указанном в свойстве «storageErrorFiles».
 
Ваш код обработки файла написан в "processFile". Не забывайте о многопоточной работе и работе внутри кода с объектами репозитория, а не с основными переменными кода!
 
Код обработки использует объект «it» с полями:
 
  • attr - обработка файловых атрибутов
  • file - файл дескриптор обработанного файла
  • storageProcessedFiles - исходный файл для хранения обработанных файлов
  • storageErrorFiles - исходный файл для хранения обработанных ошибок файлов
  • result - результат обработки файла (можно указать значение «completeResult», «errorResult» или «skipResult»)
  • errorText - текст ошибки, почему файл не был обработан
  • errorFileName - имя файла, в который будет записан текст ошибки (по умолчанию берется имя обрабатываемого файла и добавляется ".error.txt")
     
     
def t = h2table { tableName = 'story_processing' }
def s = files { rootPath = '/data'; useStory t }
def d = hdfs { rootPath = '/user/demo/data' }
csvTemp('csvfile', true) {
  header = true
  field('id') { type = integerFieldType }
  field('name') { length = 50 }
}
fileProcessing(s) {
  useSourcePath {
    mask = '{dir}/{subdir}/file_{date}.{num}'
    variable('date') { type = datetimeFieldType }
    variable('num') { type = integerFieldType }
  }
  countOfThreadProcessing = 4
  threadGroupColumns = ['date']
  order = ['num']
  removeFiles = true
  removeEmptyDirs = true
  storageProcessedFiles = d
  storageErrorFiles = d
 
  processFile {
    logFine "Processing file ${it.attr.filepath}/${it.attr.filename} ..."
    csvTemp('csvfile') {
      currentCsvConnection.path = it.file.parent
      fileName = it.file.name
      assert rows().size() > 0
    }
  }
}
